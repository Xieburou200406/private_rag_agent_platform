# 纠正：你说的是 `deploy` 目录（不是delay），完整目录结构+每个文件夹/文件用途
## 完整目录树
```
private_rag_agent_platform/
├── cpp_gateway/        # C++ epoll网关源码
├── backend_django/     # Django后端源码
├── agent_engine/       # LLM多智能体引擎源码
├── docs/               # 架构、接口、运维文档
└── deploy/             # 【部署根目录，所有容器配置集中存放】
    ├── docker-compose.yml  # 整套服务容器编排主文件
    ├── .env.example        # 环境变量模板（开源提交）
    ├── .env                # 本地私有配置（.gitignore屏蔽，不上传Git）
    ├── logs/               # 全服务运行日志持久化目录
    │   ├── gateway/        # C++网关并发、耗时、限流日志
    │   ├── django/         # Django业务、鉴权、Redis操作日志
    │   └── agent/          # LLM调用、RAG检索、多智能体执行日志
    ├── nginx/
    │   └── nginx.conf      # Nginx反向代理、动静分离、鉴权头透传配置
    ├── init_sql/
    │   └── init.sql        # MySQL初始化建表、基础数据脚本
    └── redis/
        └── redis.conf      # Redis缓存/任务队列专属配置文件
```

## 逐个拆解 deploy 下每一项作用
### 1. 根文件
#### `docker-compose.yml`
整套RAG四层平台的容器编排核心文件，定义7个服务（Nginx/C++网关/Django/Agent/MySQL/Redis/Milvus）、私有网桥、数据持久卷、启动依赖、健康检查，一条命令`docker compose up -d`一键拉起所有服务。

#### `.env.example`
无密钥的配置模板，提交到GitHub给他人参考；分类存放网关、Django、LLM向量库、评测四类参数。
#### `.env`
本地复制`.env.example`生成，填入真实数据库密码、LLM API密钥，`.gitignore`屏蔽，防止密钥泄露到代码仓库。

### 2. `logs/` 日志文件夹
统一归集所有服务运行日志，不和业务代码混杂：
- `logs/gateway`：记录epoll连接数、QPS、向量检索RTT耗时，用于性能调优（42%耗时优化数据来源）；
- `logs/django`：用户登录、对话记录、数据库操作、Redis缓存读写日志；
- `logs/agent`：文档分块、向量召回、LLM生成、Function Calling多智能体执行日志，用于离线评测准确率统计。

### 3. `nginx/` 流量入口配置目录
存放Nginx专属配置`nginx.conf`：
- 区分静态资源（/static、架构图/img）直接本地返回；
- 所有动态API转发自研C++网关；
- 透传Authorization鉴权Token，实现双层流量校验；
- 配置LLM长文本生成超时、IP限流基础规则。

### 4. `init_sql/` 数据库初始化目录
`init.sql`存放全量建表语句（用户表、知识库表、对话日志表、智能体任务表），MySQL容器第一次启动自动执行，无需手动进数据库建表，保证多环境数据表结构完全统一。

### 5. `redis/` KV存储配置目录
`redis.conf`自定义Redis运行规则：
- 内存淘汰策略（热点向量缓存自动清理）；
- 会话过期时长、最大连接数；
- 异步文档任务队列相关参数；
所有Django读写Redis共用这一套配置。

## 设计核心优势（面试可口述）
1. **配置与业务代码完全分离**：C++、Python源码放在仓库根目录，所有部署、容器、中间件配置收敛到`deploy`，代码目录干净；
2. **分层隔离**：Nginx、Redis、MySQL脚本各自独立文件夹，修改流量规则/缓存策略不用改动compose主文件；
3. **数据与日志统一归集**：日志集中在deploy/logs，方便离线评估脚本批量读取性能、效果指标；
4. **安全规范**：公私环境配置分离，真实密钥不提交GitHub；
5. **一键复现环境**：新人克隆项目，复制.env模板、执行compose命令即可完整拉起四层RAG系统。
