开发规范文档补充：Git提交规范、C++/Python命名规范、配置统一管理规则
可直接复制粘贴到 docs/03_开发规范.md
一、Git 标准化提交规范（工业级团队通用，面试加分）
1. 提交前缀分类（固定type，强制使用）
feat: 新增功能模块
fix: 修复Bug、接口异常、并发崩溃
optimize: 性能优化（C++网关提速、RAG准确率优化、缓存优化）
docs: 更新架构/接口/部署文档、README、架构图
refactor: 代码重构，无功能变更
test: 新增压测脚本、单元测试、评测实验脚本
deploy: 调整Docker、Nginx、Shell部署配置
style: 仅调整代码格式、缩进、注释，逻辑不变
2. 标准提交格式模板
<type>: 简短描述（中文，控制50字内）
空一行
可选补充：详细修改内容、优化指标、改动模块
3. 项目示例（贴合你的仓库）
feat: cpp_gateway 实现epoll分流逻辑，支持向量库直连
optimize: 优化向量检索转发逻辑，接口耗时降低42%
docs: 完善四层架构图，补充全链路LLM调用点位
fix: 修复多Agent Function Calling参数传递异常
deploy: 更新docker-compose，统一.env环境变量配置
4. 强制约束
禁止一次性提交全仓库代码，按模块拆分多次提交；
提交描述必须明确修改目录（cpp_gateway / backend_django / agent_engine）；
不提交编译产物、镜像、日志、模型权重、本地.env密钥文件（.gitignore统一拦截）。

二、C++ 代码命名规范（cpp_gateway 模块专用）
1. 文件/目录命名
全小写，下划线分隔：epoll_server.cpp、route_shunt.hpp、conn_pool/
2. 类名
大驼峰 PascalCase：ConnectionPool、RouteShunt、TrafficControl
3. 函数/方法
小驼峰 camelCase：startEpollServer()、vectorSearchForward()、calcRtStat()
4. 变量
全小写下划线：client_fd、max_qps、total_rt_ms
全局变量加前缀 g_：g_gateway_config
5. 常量/宏
全大写下划线：MAX_CONN_NUM、VECTOR_SKIP_DJANGO_FLAG
6. 注释规范
头文件、类、核心函数头部写功能注释，标注作用、入参、返回值；
性能优化关键代码段注释优化思路（如epoll分流降低42%耗时逻辑）；
7. 工程约束
CMakeLists统一管理编译配置，禁止gcc/g++硬编码编译命令；
网络、限流、压测逻辑分离子文件，单一文件不超过800行；

三、Python 代码命名规范（backend_django / agent_engine / evaluation_module，遵循PEP8）
1. 文件/目录
全小写下划线：rag_pipeline.py、multi_agent/、metric_calc.py
2. 类名
大驼峰 PascalCase：RagCore、DatasetEvaluator、LLMClient
3. 函数/变量
小写下划线：query_expand()、calc_precision_recall()、dialog_dataset
4. 常量
全大写下划线：DEFAULT_CHUNK_SIZE、LLM_TIMEOUT_SEC
5. Django 专用补充
Model表名：小写下划线 dialog_record、doc_metadata
DRF序列化器：DialogRecordSerializer
视图View：ChatQueryView
6. LLM/Agent模块额外约束
所有LLM调用封装统一工具类，禁止分散写API密钥、超时参数；
7. 代码格式
统一使用black自动格式化，缩进4空格，行宽限制120字符。

四、配置统一管理规则（核心工业化规范，杜绝硬编码）
1. 统一存放路径
所有环境、服务、密钥、限流、向量库、LLM参数全部集中存放：
deploy/.env 全局环境变量配置文件
各模块内置配置只读类，读取.env，禁止代码内写死IP、端口、密钥、阈值。
2. .env 配置分类（固定分组，与 deploy/.env.example 一致）
```env
# 1. 网关 C++ 服务配置
GATEWAY_LISTEN_PORT=8080
GATEWAY_MAX_QPS=1000
VECTOR_SKIP_PROXY=true

# 2. Django 后端配置
DJANGO_LISTEN_PORT=8000
DJANGO_SECRET_KEY=xxx
MYSQL_HOST=mysql
MYSQL_PORT=3306
REDIS_HOST=redis
REDIS_SESSION_EXPIRE=7200

# 3. Agent 引擎 & LLM & 向量库配置
AGENT_LISTEN_PORT=8001
LLM_API_KEY=xxx
LLM_TIMEOUT_SEC=30
MILVUS_HOST=milvus
MILVUS_PORT=19530
DEFAULT_TOP_K=5
CHUNK_MODE=llm_semantic

# 4. 评测实验配置
TEST_BATCH_SIZE=200
TARGET_ACC_UP=28
```
3. 分层读取规则
C++网关：读取.env，解析为全局配置结构体，启动时一次性加载；
Django后端：使用python-dotenv加载.env，settings全局引用；
Agent引擎：统一LLM工具类读取.env模型密钥、超时参数；
评测模块：从.env读取批量实验参数，无需修改代码切换实验变量。
4. 安全规范
.env 加入 .gitignore，禁止上传密钥、API Key到GitHub；
提供 deploy/.env.example 模板文件，仅保留参数名，隐藏真实密钥；
端口、并发阈值、向量检索TopK等业务参数全部外置，改配置无需重编译/重启代码；
5. 容器联动规则
在 deploy/ 目录执行 docker compose；compose 通过 env_file: .env 注入环境变量，所有容器共享一套配置。
6. 禁止行为（硬性规范）
❌ 禁止代码硬编码：LLM密钥、数据库IP、端口、QPS限流、向量库地址；
❌ 禁止分散配置：每个模块单独写配置文件，统一收敛到deploy/.env；
❌ 禁止生产环境写死实验参数，所有评测变量从环境读取。
